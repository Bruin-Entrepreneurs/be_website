Proper coding work flow

  • When you develop on one part of the website, write code on a separate branch. After finishing your part, submit a pull request and ping me so that I can look it over and then merge it into master.
  • If you're having trouble, @ message me on slack or call me on my cell. We need to start using Slack for what it's supposed to be used for.
  • When you make a new branch for yourself, create a remote branch from origin and follow the branch naming conventions -- (first-initial)(last-initial)/(task-name) For example, I would make a branch like lc/main-page
  • I'll write some more docs for the tech stack as we start to develop/design more.

Task Delegation

  • Hao - Home Page (Top Half) / Includes nav bar down to the quotes
  • Anshul - Backend -> routes, mvc, email integration, etc.
  • Mihir - Any designs you want
  • Martin - Home Page (Bottom Half) / Everything after Hao's part
  • Luke - Whatever is left
